# Solo Gainz - Project Guidelines for AI Models

> [!IMPORTANT]
> **MANDATORY RULES FOR ALL AI AGENTS WORKING ON THIS PROJECT**

## 📱 Platform Focus
*   This project is strictly a **Mobile Application** (Android & iOS).
*   Any changes or new features must be optimized for mobile interactions and performance.
*   Ignore any web or desktop platform requirements unless explicitly instructed.

## 🎨 Design & Structure Integrity
*   **Preserve Structure**: Do not remove, rename, or fundamentally change the project's folder structure or architectural patterns.
*   **Aesthetic Consistency**: The design language is high-premium, immersive, and game-like. Never deviate from the established theme defined in `lib/theme/theme.dart`.
*   **Main Design**: If creating new widgets or screens, they must feel like an organic part of the existing "Solo Gainz" world. Use consistent gradients, blurs (glassmorphism), and animations.

## 🧠 AI Intelligence & Proactivity
*   **Proactive Auditing**: You are expected to be smart and thorough. Do not just follow the user's instructions blindly. 
*   **Fix Logic & Bugs**: If you encounter a logic flaw, a performance issue, or a bug while working on a task, **you must fix it proactively**, even if it wasn't explicitly mentioned in the request.
*   **Clean Code Standards**: Always resolve compiler warnings, fix deprecated member usage (e.g., use `.withValues()` instead of `.withOpacity()`), and ensure asynchronous code is guarded by `mounted` checks.

## 📄 Project Background
*   **Name**: Solo Gainz
*   **Goal**: A unique hybrid of a fitness tracker and a combat/dungeon game.
*   **Key Features**: Frame-by-frame combat physics, immersive dungeon navigation, and high-fidelity UI.
